# `android/` — Android Native Project

This folder contains the Android native project generated by Capacitor. Most of the time you do not need to edit anything here directly — the web app in `src/` is what you work on.

## What is Capacitor?

Capacitor takes your web app (HTML, CSS, JavaScript built by Vite) and wraps it in a native Android app. The web app runs inside a `WebView` component — essentially a browser embedded in the Android app. Capacitor provides JavaScript-to-native "plugins" that let the web app access device features like the filesystem, camera, and preferences storage.

## Structure

```
android/
└── app/
    └── src/
        └── main/
            ├── AndroidManifest.xml
            └── java/com/centralcommand/app/
                └── OAuthCallbackPlugin.kt
```

## `AndroidManifest.xml`

The Android app manifest — declares the app's package name, required permissions (internet access, etc.), and registers app components. Capacitor generates and manages most of this file automatically.

## `OAuthCallbackPlugin.kt`

This is the one piece of custom native (Kotlin) code in the project. It is a Capacitor plugin that handles the OAuth redirect callback for Google sign-in on Android.

**Why is native code needed for OAuth?**

On mobile, after the user approves Google access in their browser, Google needs to redirect back to the app. It does this via a custom URL scheme like `centralcommand://oauth?code=XXXXX`. Android needs to be configured to intercept URLs with that scheme and route them to the correct app.

`OAuthCallbackPlugin` registers that custom URL scheme and exposes a JavaScript method that the `src/mobile-bridge/google-oauth.ts` code calls to wait for the redirect callback:

```
1. Web app calls api.google.connect(...)
2. mobile-bridge opens Google's auth URL in a browser (via @capacitor/browser)
3. User approves in browser
4. Google redirects to centralcommand://oauth?code=XXXXX
5. Android intercepts this URL and routes it to OAuthCallbackPlugin
6. OAuthCallbackPlugin fires a Capacitor event back to JavaScript
7. mobile-bridge receives the code and exchanges it for tokens
```

## When to edit files here

Normally never — Capacitor manages this directory. The exceptions are:
- Adding a new Capacitor plugin (run `npx cap sync android` after `npm install`)
- Modifying `OAuthCallbackPlugin.kt` if the OAuth URL scheme needs to change
- Android-specific configuration like app icons or signing keys (via `android/app/build.gradle`)

After any change to the web app, run `npm run mobile:sync` to rebuild and sync the latest web assets into this Android project.
